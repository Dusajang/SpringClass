/customer/notice.htm 요청URL p.505 Chapter 12 스프링 트랜잭션 ***** [ 격리레벨 설명(격리성, 격리수준) ] 트랜잭션 처리 과정에서 두 개 이상의 트랜잭션이 서로 동시에 같은 자원을 접근할 때에 어떻게 각각의 트랜잭션을 처리해야 하는가? 공유자원 == 같은 자원 [실습] NoticeDao 인터페이스 ㄴ hitUp() 조회수 증가 ㄴ getHit() 조회수를 얻어오는 메서드 [용어 이해] 1) Dirty Read 상황 {Dirty Read 방지: Read-Committed 이상} 정의: 한 트랜잭션에서 다른 트랜잭션이 아직 커밋하지 않은 데이터를 읽는 상황. 이로 인해 데이터의 일관성이 깨질 수 있음. 2) Non-repeatable Read 상황 {Non-repeatable Read 방지: Repeatable-Read 이상} 조건) 반드시 반복적(2번 이상)읽기 정의: 같은 트랜잭션 내에서 동일한 데이터를 두 번 조회했을 때, 중간에 다른 트랜잭션이 데이터를 수정 또는 삭제하여 결과가 달라지는 상황. 3) Phantom Read 상황 {Phantom Read 방지: Serializable} 조건) 여러 개의 레코드를 한 번에 읽어오는 작업을 반복한다. 정의: 한 트랜잭션에서 동일한 조건으로 데이터를 조회했을 때, 중간에 다른 트랜잭션이 데이터를 삽입 또는 삭제하여 결과 집합이 달라지는 상황.
